# GIT STYLE GUIDE

## Commits

1. DO follow The Conventional Commits specification
2. DO use one of "feat", "fix", "chore", "refactor" for the commit type
3. DO use scope when necessary: `<type>[optional scope]:`
4. DO write longer descriptive text on the optional body
5. DO write atomic commits
6. DO write short, efficient commit messages
